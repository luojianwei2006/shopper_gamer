# 界面原型设计说明 · 《合成小店长》

> 配套文件：`界面原型_合成小店长.html`（可点击原型，浏览器直接打开）
> 版本：v0.6-draft｜用途：定布局 / 流程 / 交互 / 层级，**非最终美术**（最终美术走 Fab 免费 UI 资源）

## 1. 设计上下文（Design Context）

| 项 | 内容 |
|----|------|
| 项目 | 合成小店长 — UE5 3D 休闲 / IAA / 边玩边赚 |
| 目标用户 | 碎片时间休闲玩家，泛年龄，移动端竖屏为主 |
| 核心场景 | 合成升级、看广告拿增益、攒积分换卡券 |
| 品牌调性 | **玩具般俏皮 (toy-like playful)** · 温暖糖果色 · 低多边形卡通 · 强反馈 |
| 竞品参照 | 合成类（Merge Mansion / 合成大西瓜）+ 网赚休闲（阳光养猪场式激励弹窗） |
| 设计禁忌 | 不用纯黑纯白、不用科技蓝+发光、不用 Inter/Roboto 默认字体（避 AI 味） |

## 2. 设计 Token（交美术/工程落地的统一规范）

### 色彩（暖调糖果色板）
| 角色 | 色值 | 用途 |
|------|------|------|
| 奶油底 `--cream` | `#FFF7EC` | 页面/面板底色 |
| 主橙 `--sun` | `#FF9F43` | 主按钮、强调、Lv 标识 |
| 主橙深 `--sun-deep` | `#F47B20` | 按钮 3D 底边、价格数字 |
| 青绿 `--mint` | `#2ED9B6` | 次按钮、任务卡、成功态 |
| 莓粉 `--berry` | `#FF6B9D` | 强调点缀、奖励图标 |
| 金币 `--gold` | `#FFC93C` | 软货币（可花金币） |
| 葡萄紫 `--grape` | `#8B5CF6` | **积分（可兑换代币，边玩边赚）** |
| 墨色 `--ink` | `#4A3526` | 文字（暖棕，非纯黑） |

### 字体
- **系统圆体栈（已锁定，零授权风险）**：`ui-rounded` / `PingFang SC` / `Microsoft YaHei` / `Hiragino Sans GB` / `system-ui`
- 展示 / 数字与正文统一走同一圆体栈（iOS/macOS 走 SF Rounded，Windows 走微软雅黑圆体感，安卓走系统圆体）
- 不再外链 Fredoka/Nunito：规避商用授权风险，真机打包直接命中系统字体，包体更小
- 兜底：若目标设备安装无圆体，则由 `system-ui` 接住，不破版

### 形状 / 阴影
- 圆角：大卡 28px / 中卡 20px / 小元件 14px（统一"软糖"感）
- 按钮：5px 实色底边模拟按压 3D，点击下沉 4px
- 阴影：暖棕半透明 `0 10px 24px -8px rgba(120,80,30,.28)`（非冷灰）

## 3. 屏幕清单与要点

| # | 场景 | 核心决策 | 关键交互 |
|---|------|---------|---------|
| ① | 启动封面 Splash | 品牌第一印象：工作室 LOGO + 游戏名《合成小店长》；点击进入 | 点击任意处 → Loading（对应 GDD §17.1） |
| ② | Loading 资源 | 异步加载进度条 + 小贴士；完成后进账号 | 自动跑进度，满 100% → 账号 |
| ③ | 账号 Account | 登陆 / 注册 / 游客 三模式切换 | 登陆走 HTTP `/auth/login`（§14）；游客本地 SaveGame；→ 游戏菜单 |
| ④ | 游戏菜单 Menu | 游戏"大厅"：开始游戏 / 游戏说明 / 退出游戏 | 开始 → 店铺实景；说明 → 帮助层；退出 → 封面 |
| ⑤ | 店铺实景 Shop（世界） | **唯一游戏内场景**：45° 室内真 3D 卡通 + 黑色阴影光照揭示；功能入口从底部 Tab 改为 **HUD 头顶（签到📅 / 设置⚙️）+ 场景左右悬浮栏**（左：合成🧩·订单📋；右：广告📺·商城🎁·任务✅·成就🏆） | 点空座位花 100 金币招店员（队列裂列）；招厨师提产能；模拟等待演示怒气/信誉（对应 §16）；左右栏 / 头脑顶开模态层；签到弹层领 7 日资源 |

**模态功能层（Overlay · 去 App 化关键）**：合成（拖拽演示）/ 订单·邻里（含好感招呼）/ 广告（S2S 校验）/ 商城（阶梯兑换）/ 任务（每日·每周·章节）/ 成就（铜银金）/ 设置（音效·音乐·语言·退出登录）。均在 3D 场景上弹出，非整屏切换，保持"我在店里操作"的临场感。

**流程串联（游戏感）**：①封面 → ②载入 → ③账号 → ④菜单 → ⑤店铺世界（含全部玩法模态层）。这正是修复"像 App 不像游戏"的核心——完整开场仪式 + 单一世界场景承载玩法（对照 GDD §17）。

> ⑤与模态层用 CSS 伪 3D（旋转地面 `rotateX(56deg)` + 黑色阴影遮罩 `radial-gradient` 渐暗）演示"45° 室内 + 未被灯光覆盖处为黑"的**信息结构与交互层级**，非 UE5 实机画面；真机用《免费资源清单_Fab.md》的 Cozy Cartoon Interior 卡通室内包 + Anime Low Poly / 17 Toon Characters 角色（套 Mixamo 动画）替代，并用 UE 点光源/聚光灯实现 §17.2 光照揭示（扩张时移动光源 + 实例化新区域）。点座位用 UMG 确认气泡防误触小模型（对应 §15.2 点击精度）。

## 4. Fab 免费 UI 资源替换映射

原型里标注「Fab」角标的元件，最终用《免费资源清单_Fab.md》中的免费包替换：

| 原型元件 | 建议 Fab 资源 | 说明 |
|---------|--------------|------|
| 头顶签到/设置按钮 · 左右悬浮栏入口 | Orange Casual GUI / Flat Cartoon GUI 的圆形 button + 侧边 dock | 头部为圆形按钮，场景两侧为竖排悬浮按钮（glass 风），避免自绘 |
| 按钮（主/次/ghost） | 同上 GUI 包的 button 组件 | 保留 3D 底边样式，换贴图 |
| 弹窗 bottom-sheet | Basic Casual GUI 的 panel/popup | 圆角面板直接用 |
| 资源 chip / 进度条 | GUI6600 或 Casual GUI 的 progress/coin | 货币图标可换 Fab 金币/钻石 Sprite |
| 卡片/面板底 | 任意 Casual GUI 的 card 组件 | 统一圆角质感 |

> 注意：Fab 包多为 Sprite/Texture，需美术切图后由 UMG 重新拼装；原型仅定"槽位与层级"，不绑定具体贴图。

## 5. 评审检查点（决策状态）

> 原 4 项检查点本轮已全部拍板，标注 ✅ 已决：

1. ✅ **合成手势 = 拖拽（已锁定）**：GDD §4.1 明确标注"设计已锁定：合成手势 = 拖拽，不采用纯点击"；原型⑤模态层"合成"用指针事件实现真·拖拽（鼠标/触屏通用），并保留「再演示一次」重置。影响 UE5 输入层（拖拽射线拾取 + 磁吸高亮）与合并动效。
2. ✅ **激励视频触发点已扩展为 9 类**：GDD §5.1 表格列出（订单双倍 / 生产加速 / +1 合成格 / 每日宝箱 / 离线收益翻倍 / 离线收益领取 / 顾客加急配送 / 信誉救场 / 章节卡点加速），均为主动点击 + 服务端 S2S 校验发奖；`DailyAdCap` 为日总上限，不因触点多而放宽单触点频率。
3. 🔲 **兑换首兑门槛可达性**：当前经济表 §3 为 ¥5 话费 = 5000 积分（原"200 积分"表述已作废）。需在纸面模拟 §6 跑小鱼路径，确认首周能否凑到门槛 ~60%（余下靠经营补足）；经济表 §8 的 `AffinityFirstRedeemBoost`(满好感一次性 500 积分) 即"首兑助推"设计。
4. ✅ **真机字体 = 系统圆体**：原型已移除 Fredoka/Nunito 外链，改 `ui-rounded / PingFang SC / Microsoft YaHei` 栈，零授权风险、包体更小。
5. ✅ **去 App 化 · 完整游戏流程**：已按 GDD §17 做出 5 幕场景（封面 / Loading / 账号登陆·注册·游客 / 游戏菜单开始·说明·退出 / 店铺实景），店铺实景为唯一世界场景承载全部玩法；功能入口已从底部 Tab 改为 **HUD 头顶（签到/设置）+ 场景左右悬浮栏**，模态层（合成/订单/广告/商城/任务/成就/设置/签到）打开，非整屏切换；45° 室内真 3D 卡通 + 黑色阴影光照揭示已在原型中可视化（§17.2）。任务系统（§18）与成就系统（§19）均已在原型做出可交互 demo。每日签到领取资源（§8 留存 / 上一轮需求）已在原型做出 7 日签到弹层 demo。
