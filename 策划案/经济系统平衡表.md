## 5. 数值推导方法（不靠拍脑袋）

1. **从 LTV 倒推汇率**：`单用户广告 LTV` ÷ `目标兑换成本率(如 30%)` = 可发放积分总量。
2. **订单奖励锚定进度**：让"看广告翻倍"后的订单积分 ≈ 该产业链解锁成本的 10%–15%，保证节奏。
3. **纸面模拟 3 类玩家**（蒙特卡洛可选）：
   - 小鱼（只挂机、几乎不看广告）：靠时间也能缓慢推进，但兑换极慢 → 保留希望。
   - 海豚（每天 2–3 局 + 看部分广告）：周内能首兑低门槛奖品 → 核心盘。
   - 鲸鱼（高频 + 全广告触点）：快速推进，需要高门槛实物作为"荣耀 sink" → 防通胀。
4. **目标曲线**：XP/店铺等级用对数曲线 `LevelCost = Base × log(n+1)`，避免后期爆炸。

## 6. 纸面模拟（首兑门槛 · v1.4 已填实）

> 目的：验证「小鱼」玩家首周能否凑到 ¥5 话费券(5000 积分)的 ~60%（§8.3 首兑助推闭环），并保证三类路径经济可解（无死循环 / 无无限刷 / 无不可达）。
> 结论速览：**小鱼 59%（保留希望）、海豚 150%（核心盘达标）、鲸鱼 297%（冲高门槛）**；聚合叙事占比 ≈ 25.8%（压线 §4 上限）。

| 假设项 | 取值 | 依据 |
|--------|------|------|
| 日均时长 | 小鱼 15min / 海豚 30min / 鲸鱼 60min | 三类画像（§5） |
| 日均广告 | 小鱼 0 / 海豚 5 / 鲸鱼 12 | 激励视频触发点（GDD §5.1 共 9 类） |
| 订单积分/单 `OrderPointReward` | 40 | 低阶商品锚定，随阶数升 |
| 广告里程碑 | 每 5 次 = 150 | 防刷，需 LTV 倒推 |
| 每日任务积分 | merge120 + order120 + ad80 = 320（全完成） | §9.1 |
| 每周任务积分 | 200/周 | 周活承接 |

### 6.2 三类玩家路径（首周 = 7 日；一次性奖励仅计 1 次）
| 玩家类型 | 日均时长 | 日均广告 | 日积分产出 | 周累计积分 | 占首兑% | 能否周首兑(5000) |
|----------|----------|----------|------------|------------|----------|------------------|
| 小鱼 | 15min | 0 | ~350 | **2940** | 59% | 否（差 2060，保留希望） |
| 海豚 | 30min | 5 | ~1073 | **7510** | 150% | 是（核心盘达标） |
| 鲸鱼 | 60min | 12 | ~2123 | **14860** | 297% | 是（富余冲高门槛） |

> **拆解（便于复盘与调参）**
> - 小鱼：订单 80×7=560 + 每日任务 100×7=700（仅做 merge/order，跳 ad 任务）+ 每周 200 + 好感 500 + 章节(Ch1+Ch2)900 + 成就 80 = **2940**
> - 海豚：订单 480×7=3360 + 广告里程碑 150 + 每日任务 320×7=2240 + 每周 200 + 好感 500 + 章节(Ch1–Ch2)900 + 成就 160 = **7510**
> - 鲸鱼：订单 1400×7=9800 + 广告里程碑 300 + 每日任务 320×7=2240 + 每周 200 + 好感 500 + 章节(Ch1–Ch3)1500 + 成就 320 = **14860**
>
> 金币侧：小鱼日金币 ~200（自售）、海豚 ~800、鲸鱼 ~2500，流向 §2.1/§7 招募与扩建 sink，本表不展开。

### 6.3 额外路径：纯剧情向玩家（验证 §8.3 防通胀）
| 路径 | 日均行为 | 周累计积分(估) | 其中叙事占比 | 结论 |
|------|----------|----------------|--------------|------|
| 纯剧情向 | 高频看广告加急 + 速通章节 | ~9000 | ≈ 35% | 叙事产出有硬上限，无法无限刷 |

> **反作弊 / 防通胀核心属性（§8.3 + GDD §14 服务器权威）**：章节数固定 10、满好感一次性积分服务端标记不可刷、成就总量有限 → **叙事总产出存在硬天花板**。即便纯剧情向玩家全力速通，积分也不会无限增长。`NarrativeShareCap`(建议 25%) 是**聚合指标**（Σ叙事/Σ总产出，需真实 DAU 占比测算）；小鱼"叙事占自身 60%"是低绝对量的边缘现象，不触发全服通胀。

### 6.4 聚合叙事占比 sanity check（假设 DAU 混合 50% 小鱼 / 35% 海豚 / 15% 鲸鱼）
- 加权周总积分 = 0.5×2940 + 0.35×7510 + 0.15×14860 ≈ **6328/人**
- 加权周叙事积分 = 0.5×1480 + 0.35×1560 + 0.15×2320 ≈ **1634/人**
- 叙事占比 = 1634 ÷ 6328 ≈ **25.8%** → 压线 §4 `NarrativeShareCap`(25%)，投放后若超则下调 `ChapterPointReward` / `AffinityFreqBonus`。

### 6.5 可解性校验（经济可解 = 无死循环 / 无不可达）
- [x] **小鱼**：首周 59%（2940），未达 5000，但第 2 周日常补足即可首兑 → 无死局、保留希望（守 P4 零挫败 / P6 不肝）。
- [x] **海豚**：首周 150% 达标首兑 → 核心盘成立。
- [x] **鲸鱼**：富余冲 ¥20 会员(18000) 与盲盒(80000) 高门槛 sink → 无无限刷、无淤积。
- [x] **叙事硬上限**：章节 10 + 好感一次性 + 成就有限 → 不存在"养小号刷好感"无限积分路径。
- [ ] **待投放验证**：广告里程碑 150/5次、每日任务 320、章节递增曲线须按单用户广告 LTV 倒推（§5 方法 1）后微调；§4 四项阈值正式定值。

### 7.4 平衡约束（防失衡）
- **产能锚定**：`厨师产能 = 厨师数 / SynthTime` ≥ `店员列数 × SpawnRate × 平均单耗`；否则出现"招了店员却永远做不完"的死局。纸面模拟验证此不等式。
- **金币闭环**：招募店员(100) + 厨师(ChefCost) 是金币主 sink，需与 §2.1 货架扩建竞争，避免金币无处花或不够花。
- **信誉软惩罚**：信誉为可恢复软惩罚，守 P4 零挫败；但长期 < 50 仍算"坏"（见 §16.6）。

### 8.3 防通胀约束（与 §4 联动）
- **叙事产出占比上限**：`NarrativeShareCap`[PLACEHOLDER]（建议 ≤25%）。即（好感积分 + 章节积分）/ 总积分产出 ≤ 25%；超则下调 `AffinityFreqBonus` 或 `ChapterPointReward`。理由：叙事是"调味"不是"主食"，占比过高会让纯经营玩家觉得被剧情玩家碾压，且冲垮兑换成本率。
- **首兑助推闭环**：`AffinityFirstRedeemBoost`(500) + 早期 `ChapterPointReward` 之和，应让**小鱼玩家**在首周也能凑到 ¥5 话费券(5000 积分)的 ~60%，剩下靠日常经营补足——保留希望但不白送（对照 §5 小鱼路径）。
- **满好感稀缺性**：每位顾客满好感一次性积分仅触发 1 次（服务端标记），不可刷；重复刷好感不给积分，避免"养小号刷好感"作弊（对照 GDD §14 服务器权威）。
- **纸面模拟**：在 §6 模板基础上，额外跑"纯剧情向玩家"（高频看广告加急 + 速通章节）路径，验证其周积分不超 `NarrativeShareCap` 反推的天花板。
